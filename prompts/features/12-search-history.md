# 12 — Add Search History with Re-run

Ensure the History tab stores every search with: (1) timestamp, (2) role + company or JD summary, (3) search query used, (4) number of results returned, (5) a "Re-run" button that re-executes the same search, (6) a "View Results" button that shows cached results if available. Sort by most recent first. Limit to last 50 searches. Each history entry should be a compact card, not a full-width row.

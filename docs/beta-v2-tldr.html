THE PITCH
SourceKit finds software engineers by analyzing their actual GitHub code contributions, not LinkedIn keywords. Enter a role, get a scored pipeline in minutes. v2.0 expands the API surface across three providers: Exa (search, enrichment, synthesis, expansion, persistent collections), Parallel (company intelligence, employee mapping, JD parsing), and the GitHub API (contributor mining, commit-level EEA scoring).

QUICK START
1. Sign in with Google at getsourcekit.vercel.app
2. Enter a role by typing role+company, pasting a JD, or dropping a job URL
3. Edit the repo list/this is the biggest lever for result quality
4. Run the search and filter by language, commits, location
5. Review candidates/try "Find LinkedIn" and "Enrich" on top matches
6. Create a Webset/set criteria and monitoring for persistent candidate tracking
7. Pipeline/add candidates, drag between stages, try bulk actions, export to CSV
PRO TIPS
Be specific
"ML Infra Engineer, training pipelines" beats "ML Engineer"
Edit repos
The AI list is a starting point. Your edits are the #1 quality lever.
Hidden gems
High commits + low followers = under-recruited and responsive
Use Websets
For recurring roles, set up monitoring to catch new contributors automatically
KNOWN LIMITATIONS
GitHub-only/use alongside LinkedIn for full coverage. Strategy takes 15-20s/expected, search is 30-60s. Some job URLs fail/paste JD text directly. LinkedIn links are AI-matched/always verify. Websets sync may take minutes/check back periodically.

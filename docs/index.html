Exa API

Search, enrichment, synthesis, expansion, and persistence. Five endpoints working together across the sourcing lifecycle.

Research API

Strategy Generation

Generates the sourcing strategy from the role spec. Runs web searches across GitHub, LinkedIn, company blogs, and job boards to identify target repos, poach companies, and EEA signals. Returns structured output with citations. No hallucinated repos or made-up star counts.

Search API

Vector Search

Matches and enriches candidates using embeddings, not keywords. Criteria can be natural language like "won or placed at a relevant hackathon" or "shipped a product in the robotics space." Matches against actual content of web pages, READMEs, and blog posts.

Answer API

Candidate Synthesis

Synthesizes a grounded summary for each top candidate. Answers "what is this person actually known for?" with web citations. Surfaces on candidate cards and feeds into outreach generation.

findSimilar API

Lateral Discovery

Click "Find Similar" on any strong candidate and get 10 more engineers with comparable work via GitHub profile similarity. Fastest path from one strong match to a full pipeline.

Websets

Persistent Pipeline

Define criteria once, Exa discovers and verifies matches, enriches them, and monitors on a schedule. Import existing lists (CSV from ATS, LinkedIn saves). Webhooks push real-time notifications when monitors find new matches.

Parallel API

Company intelligence, employee mapping, and JD parsing. Three endpoints for deep research and target company analysis.

Task API

Company Intelligence

Researches each poach company in parallel: estimated eng headcount, tech stack, recent hiring signals, attrition patterns, and a sourcing rationale. Live intelligence on every target company card.

FindAll API

Employee Mapping

Maps actual engineers at target companies by searching LinkedIn, GitHub, conference talks, and engineering blogs. Turns a static company name into a recruitable list of profiles.

Search API

JD Extraction + Research

Extracts JDs from any URL including JS-rendered pages (Lever, Greenhouse, Ashby). Does deep multi-hop research on targets for enrichment context.

GitHub API
REST + GraphQL

Contributor Mining

Mines contributors and pulls commit/contribution data from target repos. Commit volume, recency, language distribution, and repo impact scores feed directly into the 0-100 candidate scoring model.

EEA Criteria

Evidence-Based Scoring

Verifiable artifacts: papers at top venues, maintainer status on high-star repos, conference talks, shipped systems at scale, competition wins. Binary checks against public evidence, not subjective assessments.

Exa Websets

Websets are persistent, auto-updating collections of people, companies, or repos that match your criteria. Define what you're looking for once. Exa discovers, verifies, and enriches new matches on a schedule. It's a talent pipeline that runs itself.

Create a WebsetDefine your search: "ML engineers who contributed to transformer training frameworks" with criteria like "50+ commits to a top-100 ML repo" and "based in US or EU."
Discover + VerifyNeural search finds matching entities across the web. Each result gets verified against your criteria. Only verified matches make it in.
Enrich + ImportAdd fields: contact email, publications, conference talks. Import existing candidate lists (CSV from ATS, LinkedIn saves) and run the same EEA enrichment pipeline.
Monitor + NotifySet a schedule (daily, weekly). New matches get appended automatically. Webhooks push real-time notifications. Export CSV or feed into Clay and Parallel.

Usage Notes

Specific criteria matter."Senior engineer" gets noise. "50+ commits to a distributed systems repo, based in SF" gets signal.
Use findSimilar to expand from signal.One strong candidate becomes ten. Lateral discovery catches people keyword search never surfaces.
Company intel feeds outreach quality.Parallel Task data on poach targets gives you talking points. Attrition signals tell you who's reachable.
Known Limitations
GitHub-only sourcing for now (no LinkedIn, papers, or patents yet)
LinkedIn URLs are inferred and may need manual verification
JS-rendered pages may not parse correctly for JD extraction
Websets sync may take minutes; check back or set monitoring cadence
